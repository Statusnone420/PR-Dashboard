Implement fast-path for small payload serialization

account_circle Opened by core-dev-sarah
schedule 2 days ago
check_circle Open

description Issue Summary

Currently, all payload serializations allocate memory on the heap regardless of size. This creates unnecessary overhead for frequent, small messages (under 64 bytes) such as heartbeat acks and minor state updates.

We can avoid heap allocation for these payloads by utilizing a smallvec approach in the core serialization loop located in src/net/serialize.rs.

// Current
pub fn serialize(payload: &Payload) -> Vec<u8> { ... }

// Proposed
pub fn serialize(payload: &Payload) -> SmallVec<[u8; 64]> { ... }

Expected impact is a ~15% reduction in allocation overhead under load.
